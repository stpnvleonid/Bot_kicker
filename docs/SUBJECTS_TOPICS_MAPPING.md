# Предметы, топики и варианты названий в календаре

В группе Telegram используются **топики** (темы) по предметам. Названия топиков в чате и ключевые слова для сопоставления с событиями календаря приведены ниже. События из календаря сопоставляются с предметом по вхождению любого из вариантов в **название (title)** или **описание (description)** события (без учёта регистра).

---

## 1. Математика

Чтобы развести два потока математики (профиль и 100 баллов), используются **два предмета**:

- `math` — «Математика (100 баллов)»;
- `math_profile` — «Математика (профиль)»;
- `math_base` — «Математика (база)».

### 1.1. Математика (100 баллов)

**Название топика в Telegram:** `Математика (100 баллов)`

**Варианты в календаре (все упоминания для сопоставления, кроме специальных пробников/тестов и вебинаров):**

| Вариант | Примечание |
|--------|------------|
| математика | полное название |
| математике | форма косвенного падежа |
| матем | сокращение |
| мат. | сокращение с точкой |
| ЕГЭ математика | ЕГЭ |
| ЕГЭ по математике | ЕГЭ |
| ЕГЭ матем | ЕГЭ + сокращение |
| алгебра | раздел |
| геометрия | раздел |
| подготовка к ЕГЭ математика | подготовка |
| подготовка к ЕГЭ по математике | подготовка |
| разбор ЕГЭ математика | разбор |
| пробник математика | **см. поток «Математика (профиль)»** |
| пробник матем | **см. поток «Математика (профиль)»** |
| вебинар по математике | общий вебинар по математике (100‑балльный поток) |

### 1.2. Математика (профиль)

**Название топика в Telegram:** `Математика (профиль)`

**Варианты в календаре (для профильного потока):**

| профильная математика | профильная математика |

### 1.3. Математика (база)

**Название топика в Telegram:** `Математика (база)`

## Порядок проверки при нескольких совпадениях

Если в названии/описании события встречаются ключевые слова **нескольких** предметов (например, «математика и физика»), событие можно:
- привязать ко **всем** подходящим топикам, или
- привязать к **одному** по приоритету (например, по порядку: Математика, Информатика, … Английский — первое совпадение).

Рекомендация: **все подходящие топики** — тогда одно событие по нескольким предметам попадёт во все нужные топики и ко всем выбравшим эти предметы студентам.

## Реализация в коде и БД

- **Маппинг предмет → ключевые слова** вынесен в `src/config/subjects.ts`: `SUBJECT_KEYWORDS`, `detectSubjectsFromEventText(title, description)`.
- **Миграция** `migrations/002_subject_topics.sql` добавляет:
  - `group_topics` — для каждой группы и каждого предмета хранится `topic_id` (ID топика в Telegram);
  - `student_subjects` — какие предметы выбрал студент (для рассылки в ЛС);
  - `event_subjects` — к каким предметам отнести событие (по разбору названия/описания).
- **Дальнейшие шаги:** при появлении/обновлении события вызывать `detectSubjectsFromEventText`, писать результат в `event_subjects`; при отправке в чат брать `topic_id` из `group_topics` по `subject_key`; при рассылке в ЛС фильтровать студентов по `student_subjects` и `event_subjects`.
