# Переключение с тестового чата на боевой

Пошаговая инструкция: что сделать в Telegram и что прописать в БД, чтобы после тестов рассылка шла в действующий чат.

---

## Что уже есть (тестовый чат)

- В таблице **`groups`** запись с названием группы (например «2 курс»): `telegram_chat_id` = ID тестового чата, при необходимости `topic_id` или записи в **`group_topics`** (топики по предметам).
- Бот добавлен в тестовый чат, топики привязаны через `/link_topic`.
- Календарь привязан к группе через `calendar_config_id` или по имени при `/link_group`.

После переключения нужно, чтобы та же группа в БД указывала на **боевой чат** и правильные **ID топиков** боевого чата.

---

## Шаг 1. Подготовка боевого чата в Telegram

1. Создайте или откройте **боевой групповой чат** (тот, куда должны уходить события).
2. Включите **топики (темы)** в настройках чата, если нужна рассылка по предметам:
   - Настройки группы → «Темы» → Включить.
3. Создайте топики с названиями (как в боте):
   - Математика  
   - Информатика  
   - Физика  
   - Общество  
   - Русский  
   - Английский  
   (Можно не все — недостающие потом не привязывать.)
4. **Добавьте бота** в этот чат и выдайте права на отправку сообщений (и создание тем, если нужно).
5. Запомните или сохраните:
   - что это **тот же календарь** (тот же `calendar_config`), что и для теста,  
   - что админы (кто будет вызывать `/link_group`, `/link_topic`) — те же пользователи (их `telegram_user_id` уже в `admins`).

---

## Шаг 2. Узнать chat_id боевого чата

**Вариант А — через команду бота (удобно):**

1. Добавьте бота в боевой чат (если ещё не добавлен).
2. В этом чате отправьте команду: **`/chat_id`**  
   Бот ответит сообщением вида: `chat_id этого чата: -1001234567890`.  
   Это и есть ID боевого чата (сохраните его для шага 3, если будете править БД вручную).

**Вариант Б — через БД после `/link_group`:**

Если вы планируете в боевом чате вызвать `/link_group 2 курс`, то после вызова бот сам обновит `telegram_chat_id` у группы «2 курс» на ID этого чата. Тогда шаг 3 можно выполнить через команды в Telegram (см. шаг 4).

---

## Шаг 3. Переключить группу на боевой чат в БД

Нужно, чтобы запись группы (например «2 курс») указывала на **боевой** `chat_id`.

**Вариант А — обновить вручную в SQLite:**

```bash
# В корне проекта
sqlite3 .\data\bot.sqlite
```

В sqlite выполните (подставьте свои значения):

```sql
-- Посмотреть текущие группы и их chat_id
SELECT id, name, telegram_chat_id FROM groups;

-- Заменить chat_id группы «2 курс» на ID боевого чата (отрицательное число)
UPDATE groups
SET telegram_chat_id = -1001234567890
WHERE name = '2 курс';

.quit
```

Вместо `-1001234567890` подставьте реальный `chat_id` боевого чата.

**Вариант Б — через бота (без SQL):**

1. Зайдите в **боевой** чат.
2. Отправьте команду (от имени админа):  
   `/link_group 2 курс`  
   Бот обновит у группы «2 курс» поле `telegram_chat_id` на ID этого чата.  
   Если записи «2 курс» ещё нет — он создаст её и привяжет к боевому чату.

После этого все посты «в группу» будут уходить уже в боевой чат. Топики по предметам нужно заново привязать (шаг 4), потому что в новом чате у каждого топика другой `topic_id`.

---

## Шаг 4. Привязать топики боевого чата (group_topics)

В боевом чате у каждого топика свой **message_thread_id** (в коде это `topic_id`). Их нужно прописать в таблице **`group_topics`**.

**Способ 1 — через бота (рекомендуется):**

1. Зайдите в **боевой** чат (бот уже добавлен, группа уже привязана через `/link_group 2 курс`).
2. Откройте топик **«Математика»** и в этом топике отправьте:  
   `/link_topic math`  
   (или `/link_topic Математика`).
3. Аналогично для остальных предметов, заходя в каждый топик и отправляя команду:
   - В топике «Информатика»: `/link_topic informatics` или `/link_topic Информатика`
   - В топике «Физика»: `/link_topic physics`
   - В топике «Общество»: `/link_topic society`
   - В топике «Русский»: `/link_topic russian`
   - В топике «Английский»: `/link_topic english`

Бот сохранит текущий `message_thread_id` в **`group_topics`** для группы, привязанной к этому чату. Старые привязки к тестовому чату перезапишутся, так как группа одна и та же (обновился только `telegram_chat_id` в шаге 3).

**Способ 2 — вручную в БД (если знаете topic_id):**

Если вы как-то узнали `message_thread_id` каждого топика в боевом чате:

```sql
-- Узнать id группы (например «2 курс»)
SELECT id, name FROM groups WHERE name = '2 курс';

-- Обновить topic_id для каждого предмета (подставьте group_id и реальные topic_id из боевого чата)
UPDATE group_topics SET topic_id = 12345 WHERE group_id = 1 AND subject_key = 'math';
UPDATE group_topics SET topic_id = 12346 WHERE group_id = 1 AND subject_key = 'informatics';
-- ... и т.д. для physics, society, russian, english
```

Либо удалить старые привязки и вставить новые:

```sql
DELETE FROM group_topics WHERE group_id = 1;
INSERT INTO group_topics (group_id, subject_key, topic_id) VALUES
  (1, 'math', 12345),
  (1, 'informatics', 12346),
  (1, 'physics', 12347),
  (1, 'society', 12348),
  (1, 'russian', 12349),
  (1, 'english', 12350);
```

(Замените `1` на ваш `group_id` и числа на реальные `topic_id`.)

---

## Шаг 5. Проверить календарь и студентов

- **Календарь:** запись в **`calendar_config`** (и при необходимости **`groups.calendar_config_id`**) не привязана к чату — менять ничего не нужно. События по-прежнему берутся из того же календаря.
- **Студенты и предметы:** таблицы **`students`** и **`student_subjects`** привязаны к `telegram_user_id`, а не к чату. Пользователи те же — менять не нужно. Кто уже нажал `/subjects` и выбрал предметы, так и останется.
- **Админы:** таблица **`admins`** тоже по `telegram_user_id`. Если админы не меняются — ничего не трогать.

---

## Шаг 6. Отключить тестовый чат (по желанию)

Чтобы бот больше не писал в тестовый чат:

- Либо **удалите бота** из тестового чата в Telegram.  
- Либо в БД **не** создавайте и не обновляйте группу с `telegram_chat_id` тестового чата. После шагов 3–4 единственная группа «2 курс» указывает на боевой чат — в тестовый рассылка идти не будет.

Откатывать тестовые данные в БД (старые `event_chat_messages`, `send_queue` и т.п.) не обязательно; они просто перестанут использоваться для новой рассылки.

---

## Краткий чеклист

| Где | Что сделать |
|-----|-------------|
| Telegram | Боевой чат создан, топики включены и созданы (Математика, Информатика, …), бот добавлен, права на отправку выданы. |
| Telegram | В боевом чате вызвана команда `/link_group 2 курс` (или в БД вручную обновлён `groups.telegram_chat_id` на ID боевого чата). |
| Telegram | В каждом топике боевого чата вызвана команда `/link_topic <предмет>` (math, informatics, …). |
| БД | При необходимости проверить: `SELECT * FROM groups;` и `SELECT * FROM group_topics;` — группа одна, chat_id боевой, topic_id соответствуют боевому чату. |
| Студенты/админы | Ничего не меняем; календарь и пользователи общие. |

После этого все новые события и напоминания будут уходить в боевой чат и в нужные топики по предметам; в ЛС — тем, кто выбрал предметы через `/subjects`.
