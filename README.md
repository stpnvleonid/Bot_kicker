# Bot Kicker

Telegram-бот: события из Google Calendar → напоминания в чат (по топикам-предметам) и в ЛС, выборочная рассылка студентам.

## Требования

- Node.js 18+
- Токен бота от [@BotFather](https://t.me/BotFather)
- (Опционально) Google Calendar API: Service Account JSON для синхронизации календаря

## Установка

```bash
npm install
cp .env.example .env
# Отредактировать .env: BOT_TOKEN, DATABASE_URL (например file:./data/bot.sqlite)
# Для календаря: GOOGLE_APPLICATION_CREDENTIALS=путь/к/ключу.json, TZ=Europe/Moscow
```

В репозитории **нет** `.env` и JSON ключей Google — только `.env.example`. Скопируйте его в `.env` и положите Service Account в `secrets/` локально (см. `.gitignore`).

## Миграции БД

```bash
npm run migrate:dev
# или: npm run build && npm run migrate
```

После обновления репозитория снова выполните `migrate`, чтобы применились новые файлы из `migrations/`.

## Запуск и остановка

```bash
npm run dev
# или: npm run build && npm start
```

Остановка бота (из другого терминала в корне проекта):

```bash
npm stop
```

## Поведение бота

- **Календарь**: синхронизация только на **7 дней вперёд** (прошлое + неделя). События дальше не хранятся.
- **Уведомления**: нет массовой рассылки при первом запуске. Сообщения уходят **только как напоминания** — за **15 минут** и за **5 минут** до начала события (по московскому времени в тексте).
- **Топики**: напоминания в группе идут **в ветки по предметам** (Математика, Физика и т.д.), если топики привязаны через `/link_topic`. Иначе в General не шлём при наличии предметов у события.
- **ЛС**: только студентам, привязанным к группе (например через `/start` в группе) и с выбранными предметами (`/subjects`). Учитываются тихие часы из `/settings`.
- **Ссылки**: из описания/места события в календаре в сообщение подставляется первая найденная ссылка (просто URL, без разметки).
- **Планер**: в 10:00 по Москве бот предлагает в ЛС запланировать до 6 задач на день, в 20:00 — присылает задачи и просит отметить статус (выполнена/частично/отменена), а в 21:00 выгружает результаты в Google Sheets.

## Команды бота

| Команда | Кто | Описание |
|--------|-----|----------|
| `/start` | все | Регистрация; в группе — привязка к группе для ЛС. |
| `/help` | все | Список команд и краткая справка. |
| `/settings` | в ЛС | Вкл/выкл ЛС, тихие часы (22:00–08:00). |
| `/subjects` | в ЛС | Выбор предметов, по которым приходят уведомления в ЛС. |
| `/planner` | в ЛС | Краткая справка о планере учебных задач. |
| `/link_group Название` | админ, в группе | Привязать чат к группе и календарю с таким именем. |
| `/link_topic предмет` | админ, в топике | Привязать этот топик к предмету (math, физика и т.д.). |
| `/check_topics` | в группе | Показать привязки топиков; подсказка, если всё уходит в General. |
| `/chat_id` | все | Показать chat_id и topic_id текущего чата/топика. |
| `/groups` | админ | Список групп и их chat_id. |
| `/events [N]` | все | Ближайшие события (по умолчанию 10, макс. 50). |
| `/sync_now` | админ | Ручной запуск синхронизации календаря (Job 1). |
| `/select C=Имя` | админ | Выборка получателей по курсу/группе. |
| `/select E=ID` | админ | Выборка получателей по событию (кто получил бы ЛС). |
| `/select G=<предмет>` | админ | Выборка всех студентов по предмету (все, кто выбрал этот предмет в /subjects). |
| `/push Текст` | админ | Рассылка в ЛС по последней выборке (с подтверждением, лимит 60/час). |
| `/subscribers` | админ | Кто подписан на бота и календарь «2 курс». |

## Инструкция для админа

### 1. Права и календарь

- **Добавить себя в админы** (один раз):
  - Узнай свой Telegram user ID (например [@userinfobot](https://t.me/userinfobot)).
  - В БД:
    ```bash
    sqlite3 data/bot.sqlite "INSERT OR IGNORE INTO admins (telegram_user_id) VALUES (ТВОЙ_ID);"
    ```
- **Календарь**: в `calendar_config` должна быть запись с `enabled=1` и правильным `calendar_id`. Путь к ключу сервисного аккаунта задай в `.env` как `GOOGLE_APPLICATION_CREDENTIALS`. Если при старте бот пишет, что календарь не настроен — Job1 по расписанию не запустится; ручной запуск: `/sync_now`.

### 2. Привязка группы и топиков (чтобы сообщения шли в ветки, а не в General)

1. Добавь бота в группу, выдай права на отправку сообщений.
2. **В группе** выполни (название — как в `calendar_config.name` или как хочешь назвать группу):
   ```
   /link_group 2 курс
   ```
3. **В каждом топике по предмету** зайди в топик (Математика, Физика и т.д.) и в **этом топике** выполни:
   ```
   /link_topic математика
   /link_topic физика
   /link_topic русский
   ```
   и т.д. (можно ключ: math, physics, russian или название по-русски).
4. Проверка: в группе напиши **`/check_topics`** — бот покажет привязки. Если топики не привязаны, напоминания в ветки не пойдут.

### 3. Если сообщения всё равно уходят в General

- У события может быть пустой список предметов в БД. Один раз выполни дозаполнение:
  ```bash
  npm run backfill-event-subjects
  ```
- Диагностика по конкретному событию:
  ```bash
  npm run debug-routing
  npm run debug-routing 50
  ```
  (без аргумента — ближайшее событие, с числом — событие с указанным id). В выводе смотри блоки **event_subjects** и **Итоговые цели**.

### 4. Рассылка в ЛС (push)

1. Сделай выборку:
   - по группе/курсу: `/select C=2 курс`
   - по событию: `/select E=42` (id из `/events`).
   - по предмету:
     - `/select G=math` или `/select G=Математика (100 баллов)` — поток 100 баллов;
     - `/select G=math_profile` или `/select G=Математика (профиль)` — профильный поток;
     - `/select G=math_base` или `/select G=Математика (база)` — базовый поток.
2. Отправь текст: `/push Ваш текст сообщения`.
3. Подтверди кнопкой **«Да»** (лимит 60 сообщений в час на админа). Текст уходит как обычное сообщение (без разметки).

### 5. Полезные скрипты (из корня проекта)

| Команда | Назначение |
|--------|------------|
| `npm run check-calendar` | Проверка доступа к календарю и конфига. |
| `npm run debug-upcoming` | События на ближайшие 24 ч и попадание в окна 15/5 мин. |
| `npm run debug-routing [event_id]` | Почему сообщение ушло в General или в другой топик. |
| `npm run backfill-event-subjects` | Дозаполнить предметы у событий с пустым event_subjects. |

## Структура проекта

- `src/index.ts` — точка входа, cron (Job1–Job5), воркер очереди, валидация конфига календаря при старте.
- `src/config.ts`, `src/config/subjects.ts` — конфиг из env, ключевые слова предметов.
- `src/db/` — SQLite, миграции.
- `src/bot/commands.ts` — все команды и callback (settings, push confirm).
- `src/jobs/` — Job 1 (календарь, 7 дней), Job 2 (напоминания 15/5 мин, топики, тихие часы, отчёты по посещаемости), Job 3 (очередь отправки, 429 → повтор), Job 4 (очистка), Job 5 (планер учебных задач и экспорт в Google Sheets).
- `migrations/` — схема БД.
- `scripts/` — check-calendar, debug-upcoming, debug-routing, backfill-event-subjects, stop.

Дальнейшие шаги и идеи — в [IMPROVEMENTS_ROADMAP.md](docs/IMPROVEMENTS_ROADMAP.md).
